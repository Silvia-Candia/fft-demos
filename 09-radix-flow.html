9 · Radix reshapes the flow graph

Same N, same transform. Raising the radix splits the input into more piles at once, so there are fewer stages, but each butterfly is wider (an r-point block). This is the structural side of the U-shape in demo 3. (Part 1)

Reading it: each shaded block is one radix-r butterfly: r inputs in, r outputs out (a small r-point DFT). Stages = logrN, and each stage holds N/r blocks. As r grows the graph gets shorter (fewer stages) but the blocks get taller (heavier butterflies). Whether total work goes up or down is the U-shape: see demo 3 for the operation-count tradeoff. Radix-r needs N to be a power of r, which is why radix-8 is unavailable at N=16.
