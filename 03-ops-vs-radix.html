3 · Ops vs radix: the U-shape

Operation count does not keep falling as the radix grows. It bottoms out at small radix (2 to 4), which is why those are used in practice. (Part 1)

A · The shape (static)

Each series scaled to its own max, so only the shape shows. This shape does not depend on N: the radix that minimises the count is the same for every length. That is the point of the U.

B · The real cost for one N

Modelled total operation count, log scale. Pick N and read the absolute cost off the axis. The U survives, but now the whole curve sits at a height set by N.

C · All four N together

The same absolute cost for every N at once, log scale. Cost climbs steeply with N, while the U-minimum stays at the same small radix for all of them.

Model note: stages = logrN is exact. The per-butterfly multiply counts are illustrative values reflecting that high-radix butterflies need more internal multiplies, while radix-2/4 keep many trivial twiddles (multiply by ±1, ±i). The qualitative U, with its minimum at small radix, is the teaching point, not the exact integers. Radix depends only on how N factorises, never on the signal.
